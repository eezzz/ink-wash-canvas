# 缺陷复盘日志

## #1 毛笔收锋"箭头疙瘩"（b1~b13，已修复）

### 现象
毛笔模式快速拖拽抬笔时，笔画末端出现比笔身宽的箭头/疙瘩状墨块。
慢速抬笔无此问题；**水分越低箭头越大**（最终的关键线索，由用户发现）。

### 根因（b13 确认）
`inkLevel`（墨量）是一个**双重职责参数**：

1. 派生 `darkness` → 控制墨色深浅（早期已解耦）；
2. 驱动 shader 中的飞白阈值 `th = (1-inkLevel)*1.7-0.5` → 控制笔毫遮罩 mask，
   **即实际可见笔宽**。干笔时飞白从两侧吃掉笔画，可见宽度 ≪ 几何半径。

收锋代码为"聚拢笔毫防分叉"强制 `ink = max(ink, 0.75)`，使收锋段 mask 全开、
画出完整几何半径——比被飞白吃细的笔身宽一圈 → 箭头。
水分越低 → 起始墨量越低 → 笔身被吃得越细 → 宽度差越大 → 箭头越明显。
慢速抬笔走顿收分支（墨量原样传入），宽度一致，故从无问题。

修复：收锋/顿笔点的墨量原样传入，不再篡改（commit 0f54b3b，b13）。

### 为什么六轮修复都没打中

1. **心智模型错误——"笔宽 = radius uniform"**。
   实际可见笔宽 = radius × 飞白 mask 覆盖率。所有基于宽度的分析
   （宽度增长、收锋起始宽度、投影方向）都只审计了 radius 的数值链路，
   而真正的宽度差藏在 shader 遮罩里。每轮"检查宽度连续性"都显示无问题。

2. **验证工具与心智模型共享同一盲区**。
   离线仿真渲染器把 stamp 画成实心圆（mask≡1，明确跳过了飞白项），
   所以仿真**原理上不可能复现**此缺陷，反而每轮都输出"笔尾干净"，
   制造了虚假信心。教训：渲染类缺陷的仿真必须完整复刻 fragment 剖面。

3. **测试从未扫参数**。
   全部测试用默认水分 0.35，效应中等、易被其他伪影掩盖。
   用户把水分调低后效应被放大才暴露规律。教训：视觉缺陷要做
   参数矩阵（水分/浓淡/笔锋/速度 极值组合）。

4. **锚定于"看似合理"的嫌疑机制**。
   每轮都找到一个真实存在的缺陷（直线投影收锋、减速鼓宽、宽度下限平头、
   鼠标按键抖动幻影笔画）——它们都值得修，修完解释也自洽，
   于是以"解释合理"而非"先复现→修复→复现消失"作为收工标准。
   前五轮实际是在修**共存的次要缺陷**，主缺陷始终未被复现过。

5. **决定性信息来自用户的相关性观察**（水分 ↔ 箭头大小）。
   教训：视觉缺陷持续存活时，应尽早向用户收集相关变量
   （什么滑杆设置、什么速度、什么工具下更明显），而不是继续迭代修复。

### 顺带修掉的真实次要缺陷（保留）
- 收锋墨色强制加深 → 重墨锥形（darkness 解耦）
- 减速被"慢则粗"放大 → 末端鼓包（速度包络不对称 + 铺毫行程门控）
- 收锋宽度下限 0.03 → 平头小尾巴（改为收敛到零）
- 直线投影收锋脱离轨迹 → 改为尾段缓冲、沿真实轨迹收锋
- 鼠标微动开关抖动的幻影二次笔画 → 80ms 落笔去抖 + 微笔画弃墨
- pointercancel 未处理导致的状态卡死隐患

### 后续行动

- [ ] 建立参数扫描场景：水分 {0, 0.5, 1} × 速度 {慢, 快} × 收笔 {出锋, 顿收}

## #2 中文全部乱码（b15，已修复于 b16）

原因：用 Windows PowerShell `Get-Content`/`Set-Content` 整文件读写改版本号，
PS 5.1 对无 BOM 的 UTF-8 文件默认按 ANSI 解码，写回即乱码，且被提交推送。
修复：`git checkout` 回退到最后的好版本，改动一律经由编辑工具重新应用。
规则：**永远不用 PowerShell 整文件读写源码**；只用编辑工具改文件，
PowerShell 仅做 git/构建/校验（校验读取用 `[IO.File]::ReadAllText`）。

## #3 收笔僵硬（b17~b21 反复调形无效，b22 重构解决）

历代方案都在"抬笔后把已缓冲的笔尾重新渲染成收锋形状"——无论怎么调
收敛曲线/贴边/动画，本质都是**改形**，所以僵硬。查书法运笔原理后确认：
"出锋是笔行尽处，笔锋高提离纸，锋出笔画之外"；露锋收笔要"边走边提，
运笔一次比一次轻，直到写出笔锋"——锋是**继续行笔+逐渐提压写出来的**。

b22 重构：删除尾段缓冲/预览层/收锋动画/快慢分支全套机制；行笔即写即
落墨（零延迟）；抬笔启动 ~150ms"提笔相"物理——以抬笔瞬间的笔速/笔宽/
曲率继续行进，笔压连续衰减到零，途中照常落墨。快提自然写出长锋，
停顿抬笔行程趋零自然圆收，全部行为出自同一物理，无分支无重渲染。

## 验证流程（b18 起强制执行）

每次笔触相关改动，交付前必须走完自动闭环，不得让用户人工回归测试：

1. `node --check` 语法 + smoke 冒烟（stub DOM/WebGL 跑完初始化路径）
2. 启动本地浏览器（`?debug` 开事件轨迹层）
3. 用系统级鼠标事件自动画三类标准笔画：快直线出锋 / 弧线甩尾 / 慢停顿收
4. 截屏自检形态，不满意就继续改，满意才 commit + push
5. 页面左上角构建号（bN）与本日志同步递增

### 通用检查清单（视觉缺陷）
1. 先复现，后修复；修复后用同一手段确认消失
2. 仿真/测试工具是否覆盖了 shader 的全部可见性项？
3. 是否有参数在同时控制多个视觉属性（双重职责）？
4. 扫参数极值，找伪影与参数的相关性
5. 向用户收集"什么条件下更明显"，相关性比猜测值钱
